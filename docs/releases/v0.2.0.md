# @brasalabs/skills v0.2.0 Release Notes

## Highlights

- Migrates active install lists from `.brasa/skills.install.json` to `.llms/skills.json`.
- Migrates remote skill marketplace discovery from `.brasa/skills.marketplace.json` to `.llms/skills.marketplace.json`.
- Uses canonical BrasaLabs schemas from `brasalabs6/schemas` for `.llms` install and marketplace manifests.
- Adds default project/global install-file resolution so `brasa-skills add` and `brasa-skills install --save` can write `.llms/skills.json` without a custom `--skills` path.
- Preserves `--skills <file>` as the explicit custom install-list override.
- Adds package artifact smoke coverage for `.llms` add/install/global dry-run flows and authenticated remote marketplace lookup.

## Compatibility Notes

This is a breaking path migration for consumers still using `.brasa/*` manifests. Move install lists to `.llms/skills.json` and skill repository marketplaces to `.llms/skills.marketplace.json` before upgrading.

## Validation

- `pnpm check:release`
- Local linked-binary end-to-end `.llms` add/install/global dry-run smoke
